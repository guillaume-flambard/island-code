# Instructor Lesson Plan: Lesson 3
## Publish Your Site Online - GitHub Pages

---

## Overview & Learning Objectives

### Workshop Overview
Students will learn to publish their websites online using GitHub Pages, making them accessible to anyone on the internet. This lesson introduces version control concepts, GitHub basics, and online publishing.

### Learning Objectives
By the end of this workshop, students will be able to:
- Create a GitHub account
- Install and use GitHub Desktop
- Create a repository (project folder)
- Upload files to GitHub
- Enable GitHub Pages hosting
- Access their live website URL
- Update their website by pushing changes

### Prerequisites
- Completed Lesson 1 (HTML) and Lesson 2 (CSS)
- Have a completed website (HTML + CSS files)
- Basic understanding of files and folders
- Email address for GitHub account

### Duration
**Total Time:** 90-120 minutes

## Teaching Guide with Talking Points

### Part 1: Welcome & Introduction (5 minutes)

**Talking Points:**
- "Today we're putting your website online!"
- "Right now, only you can see it - after today, everyone can!"
- "We'll use GitHub Pages - it's free!"

**Why Publish Online?**
- "Share with friends and family"
- "Show potential employers"
- "Access from anywhere"
- "Looks professional"

**What is GitHub?**
- "GitHub is like Google Drive for code"
- "It stores your files online"
- "GitHub Pages hosts websites for free"
- "Used by millions of developers"

**Show Example:**
- Display your live GitHub Pages site
- "See? This is hosted on GitHub Pages!"
- "Everyone can visit this URL"

---

### Part 2: Create GitHub Account (15 minutes)

**Talking Points:**
- "First, we need a GitHub account"
- "It's free - like creating a Facebook account"
- "Your username will be in your website URL"

**Step-by-Step:**
1. Go to github.com
2. Click "Sign up"
3. Choose username (explain: part of URL)
4. Enter email
5. Create password
6. Verify email

**Important Notes:**
- Username choice matters (part of URL)
- Suggest using real name or simple username
- Email verification required
- Walk around and help during signup

**Common Issues:**
- Email not received - check spam
- Username taken - suggest variations
- Password requirements - explain clearly

**While Waiting for Email:**
- Explain what we'll do next
- Show GitHub interface
- Prepare for next step

---

### Part 3: Install GitHub Desktop (10 minutes)

**Talking Points:**
- "GitHub Desktop makes it easier"
- "Like a helper app for GitHub"
- "Easier than using website"

**Step-by-Step:**
1. Go to desktop.github.com
2. Download for Mac/Windows
3. Install application
4. Open GitHub Desktop
5. Sign in with GitHub account

**Why GitHub Desktop?**
- Easier for beginners
- Visual interface
- Less typing
- Safer (fewer mistakes)

**Alternative:**
- Can use website if preferred
- But Desktop is recommended for beginners

**Check:**
- Everyone installed successfully?
- Everyone signed in?
- Help troubleshoot if needed

---

### Part 4: Create Repository (15 minutes)

**What is a Repository?**
- "Like a folder for your project"
- "Stores all your files"
- "Can have many repositories"

**Talking Points:**
- "Each website = one repository"
- "Give it a good name"
- "Name will be in URL"

**Step-by-Step:**
1. GitHub Desktop â†’ File â†’ New Repository
2. Name: "my-first-website" (or similar)
3. Choose location (Desktop or Documents)
4. Description: "My first website"
5. Don't initialize with README (we'll add files)
6. Click "Create Repository"

**Important:**
- Name should be simple (no spaces, lowercase)
- Can use dashes: my-first-website
- Location matters - remember where!

**Common Mistakes:**
- Forgetting to choose location
- Using spaces in name
- Initializing with README (can cause issues)

**Check:**
- Everyone has repository created?
- Everyone knows where it is?

---

### Part 5: Upload Files (20 minutes)

**Talking Points:**
- "Now we add your website files"
- "Copy HTML and CSS files to repository folder"
- "Then tell GitHub about them"

**Step 1: Add Files (10 min)**

**Action:**
1. In GitHub Desktop, click "Show in Finder" (Mac) or "Show in Explorer" (Windows)
2. Repository folder opens
3. Copy your HTML and CSS files into this folder
4. Go back to GitHub Desktop
5. Files should appear under "Changes"

**Talking Points:**
- "See? GitHub sees your files!"
- "Red = new files"
- "GitHub is tracking changes"

**Common Issues:**
- Files not showing - check correct folder
- Wrong folder - show them how to find it
- Files not copying - help troubleshoot

**Step 2: Commit Changes (5 min)**

**What is a Commit?**
- "Like saving a snapshot"
- "Tells GitHub: 'Save this version'"
- "Like a checkpoint in a game"

**Action:**
1. Write message: "First website upload"
2. Click "Commit to main"
3. Files disappear from "Changes"

**Talking Points:**
- "Commit message = what you did"
- "Be descriptive"
- "This is your first commit!"

**Step 3: Push to GitHub (5 min)**

**What is Push?**
- "Sends files to GitHub website"
- "Uploads to internet"
- "Makes files available online"

**Action:**
1. Click "Publish repository" (first time)
   OR "Push origin" (if already published)
2. Wait for upload
3. See progress bar

**Talking Points:**
- "Uploading your files..."
- "This might take a minute"
- "Don't close GitHub Desktop!"

**Common Issues:**
- Slow internet - be patient
- Upload fails - check internet, try again
- Files not uploading - check file size

**Check:**
- Everyone pushed successfully?
- Everyone's files uploaded?

---

### Part 6: Enable GitHub Pages (15 minutes)

**Talking Points:**
- "Now we turn on website hosting"
- "GitHub Pages = free website hosting"
- "One click and your site is live!"

**Step-by-Step:**

**Step 1: Navigate to Settings (5 min)**
1. Go to github.com in browser
2. Sign in
3. Find your repository (username â†’ repository name)
4. Click "Settings" tab (top right)

**Step 2: Enable Pages (5 min)**
1. Scroll to "Pages" in left sidebar
2. Under "Source", select "Deploy from a branch"
3. Choose "main" branch
4. Choose "/ (root)" folder
5. Click "Save"

**Talking Points:**
- "We're telling GitHub: 'Host this website'"
- "Branch = version of code"
- "Root = main folder"

**Step 3: Get Live URL (5 min)**
1. Wait 1-2 minutes
2. Refresh page
3. See URL: `https://yourusername.github.io/repository-name`
4. Click link
5. Website loads!

**Talking Points:**
- "Your website is LIVE!"
- "Anyone can visit this URL"
- "Share with friends!"

**Common Issues:**
- URL not appearing - wait longer, refresh
- 404 error - check index.html in root folder
- Settings not saving - check internet

**Celebration:**
- "Everyone should see their website!"
- "This is amazing!"
- "You published a website!"

## Common Issues & Solutions

### Issue: Can't create GitHub account
**Symptoms:** Signup fails, email not received
**Solutions:**
- Check internet connection
- Use different email
- Check spam folder
- Try different browser
- Wait and retry

### Issue: GitHub Desktop won't install
**Symptoms:** Install fails, errors
**Solutions:**
- Check system requirements
- Download again
- Try website version instead
- Admin/administrator rights needed
- Check disk space

### Issue: Repository not found
**Symptoms:** Can't find repository
**Solutions:**
- Check you're signed in
- Refresh GitHub Desktop
- Check repository name spelling
- Look in correct location

### Issue: Files not uploading
**Symptoms:** Push fails, files don't appear
**Solutions:**
- Check internet connection
- Check files in correct folder
- Try pushing again
- Check file size (not too large)
- Check GitHub status (might be down)

### Issue: Pages not working
**Symptoms:** 404 error, site not loading
**Solutions:**
- Wait 2-3 minutes (takes time)
- Check index.html in root folder
- Check file name is exactly "index.html"
- Verify Pages enabled in Settings
- Check branch is "main"

### Issue: Changes not appearing
**Symptoms:** Updated but website same
**Solutions:**
- Make sure you pushed (not just committed)
- Wait 1-2 minutes
- Hard refresh browser (Ctrl+Shift+R)
- Check you're looking at right URL
- Clear browser cache

### Issue: Forgot repository location
**Symptoms:** Can't find folder
**Solutions:**
- GitHub Desktop â†’ Repository â†’ Show in Finder/Explorer
- Check default location (Documents/Desktop)
- Search computer for repository name

### Issue: Multiple repositories confusion
**Symptoms:** Uploaded to wrong repository
**Solutions:**
- Check repository name
- Make sure you're in right folder
- Can delete wrong repository, start over
- Or create new repository for correct files

## Differentiation Strategies

### For Beginners (Need Extra Support)
- **Pre-create account:** Help before workshop
- **Step-by-step:** Very detailed instructions
- **Screen share:** Show process on projector
- **One-on-one:** Help during each step
- **Template:** Use pre-made repository
- **Skip updates:** Focus on just publishing

**Modifications:**
- Simplify to: create account â†’ upload â†’ publish
- Skip updating website initially
- Focus on success, not perfection

### For Intermediate Students
- **Independence:** Let them try first
- **Help others:** Become helpers
- **Multiple sites:** Create more repositories
- **Custom domain:** Preview advanced topics

**Extensions:**
- Learn Git basics (branches, commits)
- Use command line Git (advanced)
- Connect custom domain
- Learn about repositories organization

### For Advanced Students
- **Command line:** Use Git terminal commands
- **Branches:** Learn about Git branches
- **Collaboration:** Work with others
- **Teaching:** Help teach others

**Advanced Topics:**
- Git workflow (branches, merges)
- Collaboration on GitHub
- Pull requests
- GitHub Actions (automation)

### For Language Barriers
- **Visual:** Show screenshots of each step
- **Screen share:** Demonstrate live
- **Simple terms:** "Upload" not "push to remote"
- **Hands-on:** Let them do it with guidance
