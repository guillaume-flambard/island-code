# Contributing to Island Code for Locals

Thank you for your interest in contributing! This project thrives on community support.

---

## How You Can Contribute

### 1. Host a Workshop

**What it involves:**
- Provide space for 5-10 students
- Wi-Fi connection
- 1.5-2 hours on a weekly or monthly basis

**What you get:**
- Visibility in our communications
- Your venue featured in student projects
- Contribution to Koh Phangan's tech education

**Interested?** Email guillaume@islandcodeforlocals.com

---

### 2. Sponsor a Session

**Ways to sponsor:**
- Cover costs for materials (laptops, supplies)
- Provide refreshments for students
- Support with internet/data costs

**Recognition:**
- Thanked in communications
- Featured as a supporter
- Your business featured in student projects

**Contact:** guillaume@islandcodeforlocals.com

---

### 3. Volunteer as Instructor or Mentor

**Roles needed:**
- **Instructors:** Lead workshops (basic HTML/CSS knowledge helpful)
- **Mentors:** Support students during sessions
- **Teaching Assistants:** Help students one-on-one

**No experience needed:** We'll train you!

**Time commitment:** Flexible - weekly, bi-weekly, or monthly

**Skills helpful:**
- Basic web development knowledge
- Patience and good communication
- Willingness to learn and teach

**Apply:** Email guillaume@islandcodeforlocals.com

---

### 4. Translate Materials

**Help needed:**
- Translate workshop materials to Thai
- Translate workshop materials to Burmese
- Help with language during sessions

**What you'll do:**
- Translate markdown files
- Help students understand concepts
- Create bilingual resources

**Languages needed:**
- Thai ↔ English
- Burmese ↔ English

**Contact:** guillaume@islandcodeforlocals.com

## Code Contributions

### If You're a Developer

**Ways to contribute code:**

1. **Improve the website:**
   - Fix bugs
   - Add features
   - Improve accessibility
   - Optimize performance

2. **Create templates:**
   - Starter templates for students
   - Example projects
   - Code snippets

3. **Add workshop content:**
   - New workshop ideas
   - Additional exercises
   - Challenge problems

### Development Setup

1. Fork the repository
2. Clone your fork locally
3. Make your changes
4. Test thoroughly
5. Submit a pull request

### Code Style

- Use clear, readable code
- Add comments for complex parts
- Follow existing code style
- Test on mobile devices

## Reporting Issues

**Found a bug or have a suggestion?**

1. Check if issue already exists
2. Create a new issue with:
   - Clear description
   - Steps to reproduce (if bug)
   - Your suggestions (if feature request)

**Types of issues:**
- Website bugs
- Documentation errors
- Workshop material improvements
- Feature requests

## Code of Conduct

**Our values:**
- Be respectful and inclusive
- Help others learn
- Be patient with beginners
- Celebrate everyone's progress

**We welcome:**
- People of all backgrounds
- Different skill levels
- Various languages
- Different learning styles

**We don't tolerate:**
- Harassment or discrimination
- Negative attitudes
- Excluding others
