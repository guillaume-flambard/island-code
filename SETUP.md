# Setup Guide for Instructors

This guide will help you set up and run Island Code for Locals workshops.

---

## Before the Workshop

### 1. Prepare Your Materials

**Required:**
- Laptops for students (or ask them to bring their own)
- Stable Wi-Fi connection
- Projector or large screen (optional but helpful)
- Chargers and extension cords

**Recommended:**
- Backup internet connection (mobile hotspot)
- USB drives with template files
- Printed workshop guides (for students who prefer paper)

### 2. Set Up Your Laptop

**Install these tools:**
- **Text Editor:** VS Code ([code.visualstudio.com](https://code.visualstudio.com)) or any text editor
- **Browser:** Chrome or Firefox (latest version)
- **GitHub Desktop:** ([desktop.github.com](https://desktop.github.com)) for easier Git workflow

**Test everything:**
- Can you access GitHub?
- Does your text editor work?
- Can you view HTML files in your browser?

### 3. Prepare Workshop Files

**Download or prepare:**
- Workshop markdown files (from `workshops/` folder)
- HTML templates (from `templates/` folder)
- Example websites for demonstration

**Create sample projects:**
- A "Hello World" example
- A simple business page example
- This helps students see what they're building

## Workshop Structure

### Typical Session Flow

1. **Welcome (10 minutes)**
   - Introductions
   - Explain what we'll build today
   - Check everyone has laptops/access

2. **Teaching (60-90 minutes)**
   - Explain concepts
   - Live coding demonstration
   - Answer questions

3. **Hands-on Practice (30-45 minutes)**
   - Students work on their projects
   - Walk around and help
   - Encourage questions

4. **Show & Share (10-15 minutes)**
   - Students share what they built
   - Celebrate their work
   - Answer final questions

## Workshop-by-Workshop Tips

### Workshop 1: Hello World
- **Focus:** Getting comfortable with HTML
- **Common Issues:** Students might forget to save files
- **Tip:** Show them how to refresh the browser to see changes

### Workshop 2: Web Design Basics
- **Focus:** Making websites look beautiful
- **Common Issues:** CSS not linking properly
- **Tip:** Show them how to check browser console for errors

### Workshop 3: GitHub Pages
- **Focus:** Publishing online
- **Common Issues:** GitHub setup can be confusing
- **Tip:** Set up GitHub accounts beforehand if possible

### Workshop 4: AI Tools
- **Focus:** Using AI as a helper
- **Common Issues:** Students might rely too much on AI
- **Tip:** Emphasize AI is a tool, not a replacement for learning

### Workshop 5: Local Project
- **Focus:** Building complete projects
- **Common Issues:** Students might try to do too much
- **Tip:** Encourage simplicity and finishing one thing well

## Teaching Tips

### Keep It Simple
- Use simple language
- Break concepts into small steps
- Don't overwhelm with too much information

### Visual Learning
- Show examples, don't just explain
- Use drawings or diagrams
- Point to code on screen

### Encourage Questions
- Make it clear there are no stupid questions
- Repeat questions so everyone hears
- If you don't know, say "Let's find out together"

### Be Patient
- Everyone learns at different speeds
- Some students might need extra help
- Celebrate small wins

### Hands-On First
- Let students try before explaining everything
- Learning by doing is powerful
- Be ready to help when they get stuck

## Common Student Questions

**"What if I make a mistake?"**
- Mistakes are how we learn!
- Show them how to undo (Ctrl+Z / Cmd+Z)
- Explain that breaking things is part of learning

**"Why doesn't my code work?"**
- Check for typos
- Make sure files are saved
- Check browser console for errors
- Help them debug step by step

**"Is this too hard for me?"**
- Reassure them everyone starts here
- Break it into smaller steps
- Pair them with someone who understands

**"Can I do this on my phone?"**
- For learning: computers are better
- For viewing: phones work fine
- Explain why desktop is better for coding

## Adapting for Different Levels

### Beginners
- Extra time for setup
- More examples
- Simpler challenges
- More encouragement

### Intermediate Students
- Let them experiment
- Give them extra challenges
- Ask them to help others
- Teach them more advanced concepts

### Mixed Levels
- Pair beginners with intermediates
- Give optional advanced exercises
- Have beginner and advanced tracks
- Encourage peer teaching

---

## Language Considerations

### Teaching in English
- Use simple, clear English
- Avoid complex technical terms
- Use visual examples
- Check for understanding frequently

### Supporting Thai/Burmese Speakers
- Have translated materials ready
- Use translators (Google Translate, AI tools)
- Pair students who speak same language
- Learn key phrases in their language

### Visual Communication
- Point to things on screen
- Use gestures
- Draw diagrams
- Show, don't just tell

---

## Troubleshooting

### Wi-Fi Issues
- Have backup hotspot ready
- Download files beforehand
- Work offline when possible

### Software Issues
- Test everything before workshop
- Have backup text editor ready
- Know how to fix common problems

### Student Laptop Issues
- Have loaner laptops if possible
- Know how to help with different operating systems
- Have basic troubleshooting skills
